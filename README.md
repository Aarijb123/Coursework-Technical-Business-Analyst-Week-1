# Smart-Recovery Strategic Discovery Dossier - Week 1

Build the discovery and value-case materials that turn Legacy-Trust Bank's collections pain into a credible Phase 1 recommendation.

## What this week covers

This standalone coursework project helps you complete Week 1 of the course:
- frame the debt recovery problem clearly
- convert stakeholder evidence into Jobs-to-be-Done
- map the As-Is process and a first-pass To-Be flow
- build a transparent ROI model using sample data
- prepare a discovery dossier ready to feed Week 2

## Getting setup

1. **Download or copy this folder** into your working area.
2. **Open the folder in your editor** of choice.
3. **Choose your working tools**:
   - Excel or Google Sheets for CSV workbook tabs
   - Miro, Lucidchart, or PowerPoint for process maps
   - Word, Google Docs, or Markdown for the dossier
4. **Read `docs/README.md` first** so you understand the workflow.
5. **Review the sample data in `data/`** before editing any templates.

This project is documentation-first, so there is no code runtime requirement.

Recommended tools:
- Microsoft Excel or Google Sheets
- Miro or Lucidchart
- Word, PowerPoint, or any Markdown editor

## How to use this project

1. Read `docs/01-discovery-brief-template.md`.
2. Populate the evidence tables in `data/` and `templates/`.
3. Use `templates/miro-board-structure.md` to build your board layout.
4. Use the CSV tabs in `templates/roi-model/` as a starter workbook.
5. Write your finished Week 1 outputs into `submissions/`.

## How to check your progress

You are on track if you can produce:
- a completed discovery brief
- a stakeholder evidence table and prioritised JTBD list
- an As-Is process map with pain points marked directly on the flow
- a ranked opportunity model covering at least 5 automation ideas
- a short Phase 1 recommendation summary

## Files explained

- `data/` - realistic sample inputs for analysis
- `docs/` - step-by-step guidance and templates for written outputs
- `templates/` - reusable tables, ROI workbook tabs, and board structures
- `submissions/` - where you place your completed Week 1 deliverables

## Troubleshooting

### I do not know where to start
Start with `docs/01-discovery-brief-template.md`, then read the stakeholder notes in `data/stakeholder_interview_notes.csv`.

### The ROI model feels overwhelming
Begin with only three opportunities, make the formulas work, then expand to five or more.

### My process map looks too neat
That is common. Re-read the stakeholder notes and deliberately add the duplication, delays, and reconciliation work.

### I am not sure whether something is a JTBD or a feature
If it names a screen or tool, it is probably drifting toward a feature. Reframe it as an outcome.
